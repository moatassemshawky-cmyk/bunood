# Bunood — Project Guidance

## About Bunood

Bunood is a premium B2B construction materials marketplace built for Egypt, with plans to
expand across the Middle East.

The goal is **not** to become another e-commerce website.

The goal is to become the digital infrastructure that connects engineers, contractors,
consultants, project owners, developers, and suppliers in one trusted platform.

Bunood focuses on:

- Construction materials
- RFQ (Request for Quotation)
- Supplier discovery
- Verified suppliers
- Direct communication
- Price comparison
- Fast quotation process
- Trust and transparency

## Target users

- Engineers
- Contractors
- Consultants
- Procurement engineers
- Project owners
- Construction companies
- Building material suppliers

## Design philosophy

The website should feel:

- Premium
- Modern
- Clean
- Professional
- Fast
- Minimal
- Trustworthy

Inspired by companies like Stripe, Microsoft, Notion, Linear, and Apple.

Avoid flashy marketplace designs. Whitespace, typography, consistency, and clarity are more
important than animations.

## Development principles

Always prioritize:

- Clean architecture
- Reusable components
- Scalable code
- Maintainability
- Accessibility
- Performance
- Responsive design
- Type safety
- SEO
- Security

## Rules

**Never:**

- Redesign pages without asking
- Remove business features
- Change business logic
- Change database schema
- Install unnecessary packages
- Duplicate components
- Duplicate logic
- Create technical debt

**Always:**

- Explain your plan before making changes
- Work on one feature at a time
- Preserve existing functionality
- Verify the application still builds after changes
- Write production-quality code

## Communication

When making suggestions:

- Think like a Senior Staff Engineer.
- Think like a Product Manager.
- Think like a UX Designer.
- Explain WHY a change is beneficial.
- Suggest alternatives when appropriate.
- If uncertain, ask before making assumptions.
